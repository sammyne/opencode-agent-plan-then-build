## 实现建议

- **编辑策略**：所有任务都是对现有 markdown 文件的文本插入/修改，使用 `edit` 工具定位到精确位置进行修改
- **YAML frontmatter 不动**：所有任务仅修改 `---` 之后的 prompt 文本，不触碰头部的 YAML 配置
- **精炼原则**：参考 superpowers 的方法论但用更精炼的语言表达，避免将 superpowers 原文大段搬运。每条规则 1-2 行足够
- **保持中文**：现有 agent 文件使用中文，增强内容也使用中文
- **验证方式**：每个任务完成后通读修改区域的上下文，确认 markdown 格式正确、步骤编号连续、与前后内容衔接自然
